## Codebase Patterns
- 工作目录是 web/ (Next.js 16 App Router)。所有路径相对 web/。质量检查在 web/ 下跑: npm run typecheck / npm run test (vitest) / npm run lint。
- 本地 dev 须同跑 `npm run dev`(:3000) 与 `npm run dev:inngest`(:8288)，否则 Inngest sendEvent 在无 INNGEST_EVENT_KEY 的 dev 下静默 no-op，job 永远不执行(见 web/CLAUDE.md)。涉及 Inngest 的 story 写测试时 mock client，不依赖真实 dev server。
- DB schema 全在 web/src/lib/db/schema.ts (单文件, Drizzle pgTable)。新表加在此文件，drizzle-kit generate 生成 migration。已有 pgvector(vector(1536) customType)、pgEnum 大量使用、index/unique/primaryKey helper。
- DB client: web/src/lib/db/client.ts 导出 db。
- API route 模式(参考 api/agents/route.ts): "use server-only"; auth() 取 session; resolveUserId(email) 查 users.id; zod 校验 body; 未登录返回 401 {error:"Unauthorized"}; export const dynamic="force-dynamic"; runtime="nodejs"。
- 现有审计表 change_log(action_kind/target_type/target_id/before/after/reason/performed_by 'user'|'agent')。Executor/Evolver 的树变更与派发都写它。
- 现有预算表 prompt_log(user_id/kind/model/tokens_in/tokens_out)。Policy.checkBudget 读它。
- 现有限速: @upstash/ratelimit + @upstash/redis 已装。
- 现有 ingest: api/ingest(通用) + api/ingest/telegram/webhook + api/ingest/email。ingest_sources 表已含 source_type enum 'telegram'|'email'|'rss'|'webhook'|'api_claude'。回流复用此入口。
- 现有 Inngest functions 在 web/src/lib/inngest/functions/(compile-memo/daily-page/weave-graph/weekly-report/gap-detect/orphan-detect/schema-detect/fetch-rss)。新 function 加这里并在注册数组登记(查 web/src/app/api/inngest/route.ts 或 lib/inngest 的 serve 注册)。
- 知识图: pages(concept/source/synthesis/daily) + page_links(带时序 valid_from/valid_to) + page_sources。tree_nodes.page_id 关联到 pages 作记忆底座。
- UI 复用 web/src/components/ui (Card/Btn/SectionLabel 等)。图可视化复用 d3-force(已装 d3-drag/force/selection/zoom)，参考 Graph 相关组件。
- 不可变原则: 创建新对象/数组，勿原地 mutate。禁硬编码密钥(env 读取)。SQL 用 Drizzle 参数化。
- 安全 story(US-023..025 沙箱/终端/浏览器)的测试必须含: 无密钥注入、危险命令拒绝(rm -rf/curl/cat ~/.ssh)、SSRF 内网地址拒绝。这些是验收硬指标。

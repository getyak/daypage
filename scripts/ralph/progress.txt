## Codebase Patterns
- SPM packages are added directly to `DayPage.xcodeproj/project.pbxproj` using `XCRemoteSwiftPackageReference`, `XCSwiftPackageProductDependency`, and `PBXBuildFile` with `productRef` (not `fileRef`). Also add `packageReferences` to `PBXProject` and `packageProductDependencies` to `PBXNativeTarget`.
- Build command: `xcodebuild -scheme DayPage -destination 'platform=iOS Simulator,name=iPhone 17' build`
- Package resolution: `xcodebuild -resolvePackageDependencies -scheme DayPage`
- `DayPage.xcodeproj/project.xcworkspace` is in `.gitignore` — Package.resolved can't be committed directly.
- The `GeneratedSecrets.swift` is auto-generated by `scripts/generate_secrets.sh` at build time from `.env` file.
- CRITICAL: When adding new PBXGroup IDs to pbxproj, check ALL sections for conflicts (e.g. `A50000020` was taken by Utilities). New IDs must be unique across the entire file.
- `Color(hex:)` extension is in `DayPage/DesignSystem/Colors.swift` — available project-wide.
- New Swift files need 3 pbxproj entries: PBXBuildFile + PBXFileReference + group children listing + PBXSourcesBuildPhase listing.
- New feature directories need: create directory, add PBXGroup entry in pbxproj, add group to parent's children list.

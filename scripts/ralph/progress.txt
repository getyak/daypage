## Codebase Patterns
- GeneratedSecrets.swift is gitignored and auto-generated by scripts/generate_secrets.sh from .env at build time — edit the template in generate_secrets.sh, not the generated file
- SPM packages in project.pbxproj follow ID pattern SPM000XX: PBXBuildFile → XCSwiftPackageProductDependency → XCRemoteSwiftPackageReference; need entries in 4 places: PBXBuildFile, PBXFrameworksBuildPhase files, target packageProductDependencies, and project packageReferences
- Use `xcodebuild -resolvePackageDependencies` before `build` when adding new SPM packages
- Sentry SDK: `SentryUser` is ObjC class renamed to `User` in Swift (NS_SWIFT_NAME); always use `Sentry.User(userId:)` with full module qualifier to avoid conflict with Supabase's `User` type

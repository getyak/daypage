## 背景与产品定位

- 当前组织简介写明产品为 **Telepace AI**。
- 公开资料显示 Telepace.AI 方向包含 **AI tools for user research** / **AI survey for user research**。
- GitHub 组织公开描述中，`nexus` 仓库被描述为 **AI-powered reading assistant**，用于处理、理解和总结网页与文档内容。

因此，本 backlog 按以下产品体验链路拆解：

1. 新用户首次访问与价值理解。
2. 注册/登录与激活。
3. 创建研究项目/导入材料。
4. AI 摘要、洞察生成与可信度校验。
5. 用户研究问卷/访谈/反馈收集。
6. 结果管理、协作、导出与复用。
7. 性能、错误处理、可访问性、移动端体验。
8. 增长、商业化、留存与运营能力。

## Issue 1：官网/产品首页价值主张不够明确，用户难以在 5 秒内判断产品用途

**类型**：体验优化 / 转化率优化  
**优先级**：P0  
**影响范围**：新用户首次访问、试用转化、品牌认知

### 问题描述
用户进入产品首页后，需要立即理解：

- Telepace 是什么？
- 为谁服务？
- 解决什么痛点？
- 与 ChatGPT、Notion AI、Dovetail、Maze、Typeform、Readwise 等工具相比差异是什么？
- 为什么现在就值得开始试用？

如果首屏只展示抽象口号或功能名，新用户需要额外思考产品场景，容易直接流失。

### 优化建议
- 首屏标题使用结果导向表达，例如：`用 AI 在 10 分钟内从访谈、问卷和网页资料中提炼用户洞察`。
- 副标题明确目标用户：产品经理、UX Researcher、创业团队、市场研究人员、内容研究人员。
- 首屏加入 3 个明确收益点：节省分析时间、自动聚类痛点、生成可引用洞察。
- CTA 使用双按钮：`免费开始分析` + `查看示例报告`。
- 增加一段产品动图或交互式 demo，让用户不用注册即可理解核心流程。

### 验收标准
- 首屏在不滚动情况下展示目标用户、核心场景、核心收益和主 CTA。
- 新用户可在 5 秒内复述产品用途。
- 首页提供一个无需登录的示例分析结果入口。

## Issue 2：缺少高质量 Demo 数据，导致新用户空状态下不知道如何开始

**类型**：Onboarding / 空状态优化  
**优先级**：P0  
**影响范围**：首次激活率、产品理解、试用转化

### 问题描述
AI 工具如果首次进入后展示空白项目、空白文档或单一上传入口，用户会面临较高启动成本：

- 不知道应该上传什么材料。
- 不知道上传后会得到什么结果。
- 担心隐私或格式不兼容。
- 需要自己准备测试数据，降低试用意愿。

### 优化建议
- 提供 2-3 个内置 demo：
  - 用户访谈分析 demo。
  - 问卷反馈分析 demo。
  - 网页/文档阅读总结 demo。
- 空状态文案从“暂无数据”改为任务导向：`上传访谈记录、问卷 CSV 或粘贴网页链接，AI 将自动生成痛点、主题和可行动建议。`
- 在空状态加入 `试用示例数据` 按钮。
- 将首次任务拆为 3 步：导入材料 → 选择分析目标 → 查看洞察。

### 验收标准
- 新账号进入后可一键加载 demo 数据。
- demo 完整覆盖从输入到输出的核心流程。
- 空状态至少包含一个主操作、一个示例入口和一条格式说明。

## Issue 3：导入入口需要统一支持多种研究材料，减少用户选择成本

**类型**：功能建议 / 信息架构  
**优先级**：P0  
**影响范围**：内容导入成功率、核心功能使用率

### 问题描述
用户研究资料来源分散，常见包括：

- 访谈转写文本。
- 问卷 CSV / Excel。
- App Store / Google Play 评论。
- 客服对话。
- 网页文章。
- PDF、Notion、Google Docs、Figma 评论。

如果产品只支持单一导入方式，用户需要手动转换格式，体验割裂。

### 优化建议
- 设计统一导入中心，支持：上传文件、粘贴文本、输入 URL、连接第三方工具。
- 每种导入方式展示支持格式、大小限制、隐私说明和预计处理时间。
- 对 CSV/Excel 增加字段映射步骤，例如选择“用户反馈字段”“评分字段”“时间字段”。
- 导入失败时给出具体原因和修复建议，而不是通用错误。

### 验收标准
- 用户可在一个入口看到所有导入方式。
- CSV/Excel 支持字段映射和预览。
- 导入失败错误包含具体失败原因。

## Issue 4：AI 输出缺少证据链，用户难以信任洞察结论

**类型**：AI 可信度 / 核心体验  
**优先级**：P0  
**影响范围**：结果可信度、专业用户采纳率

### 问题描述
用户研究场景中，AI 不能只给结论。研究人员需要知道：

- 这个洞察来自哪些原始材料？
- 有多少用户提到？
- 原话是什么？
- 是否存在相反证据？
- AI 是否过度推断？

如果洞察没有引用来源，用户需要手动回查原文，AI 节省时间的价值会被抵消。

### 优化建议
- 每条洞察都展示引用证据：原文片段、来源、用户/样本编号、时间。
- 为洞察增加 `证据数量`、`影响程度`、`置信度`、`相反证据` 字段。
- 支持点击洞察跳转到原始材料对应位置。
- 对 AI 推断内容增加标识：`基于证据推断` / `原文明确提及`。

### 验收标准
- 每条 AI 洞察至少关联 1 条原始证据。
- 用户可从洞察一键跳转到原文。
- AI 推断与原文事实在 UI 上有明确区分。

## Issue 5：AI 生成过程缺少进度反馈，等待期间用户焦虑

**类型**：交互体验 / 性能感知  
**优先级**：P1  
**影响范围**：任务完成率、用户焦虑、重复提交

### 问题描述
AI 分析通常耗时较长。如果页面只显示 loading spinner，用户无法判断：

- 是否卡住。
- 大约还要多久。
- 当前进行到哪一步。
- 是否可以离开页面。

这会导致重复点击、刷新页面或放弃任务。

### 优化建议
- 展示分阶段进度：解析材料 → 清洗文本 → 主题聚类 → 生成洞察 → 生成报告。
- 展示预计剩余时间或已处理数量。
- 允许任务后台运行，并在完成后通知用户。
- 超过阈值时显示解释与建议：`文件较大，分析可能需要 2-3 分钟。你可以离开页面，完成后会自动保存。`

### 验收标准
- 长任务展示明确阶段状态。
- 用户离开页面后任务不会丢失。
- 失败后支持重试且不需要重新上传材料。

## Issue 6：错误提示过于技术化，缺少可操作修复路径

**类型**：错误处理 / 可用性  
**优先级**：P1  
**影响范围**：导入、分析、登录、支付、导出

### 问题描述
常见错误包括格式不支持、文件过大、URL 无法抓取、登录过期、AI 生成失败、网络超时等。如果只展示 `Something went wrong` 或接口错误码，用户无法自助恢复。

### 优化建议
- 错误信息结构统一为：发生了什么 → 为什么 → 用户可以怎么做。
- 对不同错误提供差异化操作：重试、重新上传、复制错误详情、联系客服、查看格式要求。
- 技术错误详情默认折叠，方便开发排查但不干扰普通用户。
- 对高频错误埋点并建立错误看板。

### 验收标准
- 所有用户可见错误均包含可执行下一步。
- 导入/生成失败支持一键重试。
- 前端记录错误场景和失败原因。

## Issue 7：缺少研究项目级信息架构，材料、洞察和报告之间关系不清晰

**类型**：信息架构 / 产品结构  
**优先级**：P1  
**影响范围**：复杂项目、团队协作、复用

### 问题描述
用户研究通常以项目为单位组织，而不是以单篇文档或单次对话为单位。若产品只展示文件列表或聊天记录，用户难以管理长期研究资料。

### 优化建议
- 建立项目结构：项目 → 数据源 → 分析任务 → 洞察 → 报告。
- 项目首页展示：目标、样本数量、关键发现、最近更新、待办事项。
- 支持给材料和洞察打标签，例如 `价格敏感`、`Onboarding`、`竞品迁移`。
- 支持跨项目搜索和复用洞察。

### 验收标准
- 用户可以创建、命名、归档项目。
- 每个洞察可追溯到所属项目和数据源。
- 项目页展示核心进展摘要。

## Issue 8：缺少“从洞察到行动”的产品闭环

**类型**：功能建议 / 工作流优化  
**优先级**：P1  
**影响范围**：产品经理、研究团队、决策场景

### 问题描述
研究工具的最终价值不是生成摘要，而是帮助团队做决策。如果洞察不能转化为需求、实验或路线图任务，用户还需要复制到其他工具。

### 优化建议
- 每条洞察支持生成：产品机会、用户故事、实验假设、PRD 草稿、优先级建议。
- 增加 `转为任务` 功能，支持导出到 Linear、Jira、GitHub Issues、Notion。
- 提供 ICE/RICE 优先级评分辅助。
- 报告结尾生成 `建议下一步`。

### 验收标准
- 用户可从洞察创建行动项。
- 行动项保留原始证据引用。
- 支持至少一种外部项目管理工具导出。

## Issue 9：问卷/调研功能需要加强问题生成、逻辑分支和结果分析

**类型**：功能建议 / 用户研究  
**优先级**：P1  
**影响范围**：AI survey、研究闭环

### 问题描述
如果产品定位包含 AI survey，需要覆盖从问卷设计到分析的完整链路。仅生成问题或收集回答不足以形成差异化。

### 优化建议
- 根据研究目标自动生成问卷大纲和问题。
- 支持问题类型：单选、多选、量表、开放题、排序、NPS。
- 支持逻辑分支和跳题。
- AI 自动检测诱导性问题、双重问题、过长问题。
- 回收后自动聚类开放题答案，提炼主题和代表性原话。

### 验收标准
- 用户输入研究目标后可生成可编辑问卷。
- 支持至少 5 种常见问题类型。
- 开放题结果自动聚类并可追溯原始回答。

## Issue 10：报告导出能力不足，影响团队传播和决策

**类型**：功能建议 / 协作  
**优先级**：P1  
**影响范围**：研究交付、团队协作、管理层汇报

### 问题描述
研究结论通常需要分享给产品、设计、研发、管理层。如果只停留在网页内，用户需要手动整理报告。

### 优化建议
- 支持导出为 PDF、Markdown、PPT、Notion、Google Docs。
- 报告模板按场景区分：高管摘要、产品机会报告、用户痛点报告、竞品分析报告。
- 支持隐藏敏感信息和匿名化用户原话。
- 支持公开分享链接，并设置权限和过期时间。

### 验收标准
- 至少支持 PDF 和 Markdown 导出。
- 导出报告保留洞察证据引用。
- 分享链接支持只读访问。

## Issue 11：协作能力不足，团队无法围绕洞察讨论和确认

**类型**：协作体验  
**优先级**：P2  
**影响范围**：团队版、付费转化

### 问题描述
研究洞察通常需要多人确认。如果缺少评论、标注、状态流转，团队会转移到 Slack/飞书/Notion 讨论，产品本身失去协作中心价值。

### 优化建议
- 支持对洞察、原文片段、报告段落评论。
- 支持 @成员、状态标记：待确认、已确认、需补充、已转任务。
- 支持收藏/置顶关键洞察。
- 支持变更历史和版本对比。

### 验收标准
- 用户可对洞察发表评论。
- 洞察有状态字段。
- 项目成员可收到 @ 通知。

## Issue 12：缺少隐私、安全和数据使用说明，降低企业用户信任

**类型**：信任建设 / 合规  
**优先级**：P0  
**影响范围**：企业用户、敏感研究资料、付费转化

### 问题描述
用户研究材料可能包含客户姓名、联系方式、商业计划、访谈录音和内部策略。用户在上传前会关心：

- 数据是否用于训练模型？
- 数据存储多久？
- 是否加密？
- 谁可以访问？
- 如何删除？
- 是否支持匿名化？

### 优化建议
- 上传入口附近展示简短隐私承诺。
- 增加数据处理说明页面。
- 支持项目级数据删除和导出。
- 支持自动识别并打码 PII 信息。
- 企业版提供 SSO、审计日志、数据保留策略。

### 验收标准
- 上传前用户可看到数据使用说明。
- 用户可删除项目及其原始材料。
- 支持至少姓名、邮箱、手机号的 PII 检测提示。

## Issue 13：AI 输出缺少可控性，用户无法按研究目标调整分析维度

**类型**：AI 交互 / 高级功能  
**优先级**：P1  
**影响范围**：专业研究人员、复杂分析任务

### 问题描述
不同研究目标需要不同分析框架。例如：

- 新用户激活：关注阻塞点和理解成本。
- 付费转化：关注价格异议和替代方案。
- 留存研究：关注长期价值和流失原因。
- 竞品迁移：关注替代工具和迁移动机。

如果 AI 只提供固定总结，专业用户会觉得泛泛而谈。

### 优化建议
- 分析前让用户选择目标：可用性测试、访谈总结、问卷分析、流失分析、竞品研究。
- 支持自定义分析问题，例如：`请重点找出阻碍用户完成首次导入的原因`。
- 输出可按维度切换：痛点、需求、机会、情绪、频次、人群。
- 支持重新生成指定模块，而不是整份报告重跑。

### 验收标准
- 分析任务支持选择目标或输入自定义问题。
- 同一批材料可生成不同分析视角。
- 用户可对单个洞察模块重新生成。

## Issue 14：缺少质量校验机制，AI 可能遗漏少数但关键的用户反馈

**类型**：AI 质量 / 研究严谨性  
**优先级**：P1

### 问题描述
AI 总结容易偏向高频主题，但用户研究中低频高影响问题同样重要。例如支付失败、隐私担忧、关键用户群无法完成任务等。

### 优化建议
- 输出中区分 `高频问题` 与 `低频高风险问题`。
- 标记异常反馈和强烈负面情绪。
- 增加 `遗漏检查`：列出未归类但可能重要的原始反馈。
- 支持人工合并/拆分主题，训练项目内分类规则。

### 验收标准
- 报告中包含低频高风险问题区域。
- 用户可查看未归类反馈。
- 用户可手动调整 AI 聚类结果。

## Issue 15：移动端体验需要优化，确保分享和轻量查看可用

**类型**：响应式 / 移动体验  
**优先级**：P2

### 问题描述
虽然复杂分析多在桌面端完成，但团队成员可能在手机上打开分享报告。如果移动端排版、表格、引用和导航不可用，会影响传播。

### 优化建议
- 报告页移动端优先优化阅读体验。
- 大表格改为卡片式展示。
- 洞察证据支持折叠展开。
- 长报告增加目录锚点。

### 验收标准
- 手机端可顺畅阅读分享报告。
- 关键 CTA 不被遮挡。
- 引用证据可展开查看。

## Issue 16：缺少全局搜索，历史洞察难以复用

**类型**：信息检索 / 知识库  
**优先级**：P2

### 问题描述
随着项目增多，历史访谈、问卷和洞察会沉淀为知识资产。如果不能搜索，用户很难复用过去研究成果。

### 优化建议
- 增加全局搜索，覆盖项目、材料、原文、洞察、报告。
- 支持过滤条件：项目、时间、标签、来源、情绪、用户群。
- 支持语义搜索，例如 `用户为什么不愿意付费？`。

### 验收标准
- 用户可跨项目搜索洞察和原文。
- 搜索结果显示来源和匹配片段。
- 支持至少项目、时间、标签过滤。

## Issue 17：缺少模板体系，用户每次都要从零配置分析和报告

**类型**：效率优化 / 模板  
**优先级**：P2

### 问题描述
不同场景有固定流程。如果没有模板，新用户不知道最佳实践，老用户也会重复配置。

### 优化建议
- 提供场景模板：用户访谈、可用性测试、NPS 分析、流失分析、竞品反馈分析、需求收集。
- 模板包含推荐导入格式、分析维度、报告结构和示例问题。
- 支持团队自定义模板。

### 验收标准
- 新建项目时可选择模板。
- 每个模板包含预设分析维度和报告结构。
- 用户可保存当前配置为模板。

## Issue 18：缺少关键产品埋点，无法判断体验优化优先级

**类型**：数据分析 / 增长  
**优先级**：P1

### 问题描述
要持续优化产品，需要知道用户在哪一步流失。建议追踪完整漏斗：访问 → 注册 → 创建项目 → 导入材料 → 分析成功 → 查看洞察 → 导出/分享 → 复访。

### 优化建议
- 建立核心事件埋点：
  - `sign_up_started` / `sign_up_completed`
  - `project_created`
  - `source_import_started` / `source_import_succeeded` / `source_import_failed`
  - `analysis_started` / `analysis_completed` / `analysis_failed`
  - `insight_opened`
  - `evidence_opened`
  - `report_exported`
  - `share_link_created`
- 建立转化漏斗和错误看板。
- 按用户类型、材料类型、文件大小、分析目标分组分析。

### 验收标准
- 能查看首次激活漏斗。
- 能查看导入和分析失败率。
- 每个高优先级体验优化都有指标验证。

## Issue 19：缺少国际化和中文语境优化，中文用户研究材料分析质量需专项验证

**类型**：国际化 / AI 质量  
**优先级**：P2

### 问题描述
如果面向中文用户或中英混合团队，产品需要在 UI、分词、情绪识别、原话引用、报告表达上适配中文语境。

### 优化建议
- UI 支持中英文切换。
- 中文开放题聚类专项评测。
- 支持中英混合材料分析。
- 报告语气支持切换：专业汇报、产品需求、简洁摘要。

### 验收标准
- 用户可切换中文 UI。
- 中文材料分析结果包含准确原文引用。
- 中英混合材料不会丢失语义。

## Issue 20：缺少定价与用量透明度，用户不清楚 AI 分析成本

**类型**：商业化 / 信任  
**优先级**：P2

### 问题描述
AI 产品常涉及额度、文件大小、token、项目数、成员数等限制。如果用户在关键操作后才发现限制，会造成挫败。

### 优化建议
- 在定价页清楚说明：项目数、数据源数量、AI 分析次数、文件大小、成员数、导出能力。
- 在导入和分析前提示预计消耗额度。
- 接近额度上限时提前提醒。
- 提供用量页，展示历史消耗。

### 验收标准
- 用户可在分析前看到预计额度消耗。
- 用户可查看当前套餐和剩余额度。
- 达到限制时给出升级或替代方案。

1. P0：价值主张、Demo 空状态、导入中心、证据链、隐私说明。
2. P1：进度反馈、错误处理、项目结构、行动闭环、问卷分析、导出、AI 可控性、质量校验、埋点。
3. P2：协作、移动端、全局搜索、模板、国际化、定价透明度。

## Codebase Patterns (carried over from Wave 1, append new findings here)
- Track ScrollView scroll position with a named `.coordinateSpace(name:)` on the ScrollView + a 1pt `GeometryReader`-backed anchor view at the end of its content that emits its `.frame(in: .named(...)).minY` via a `PreferenceKey`. Parent reads both the anchor minY and the ScrollView's visible height (from the outer GeometryReader's `geo.size.height`) to derive "near bottom" — no ScrollViewReader needed for that. Works on iOS 16.
- When removing a card from the timeline that was used as the pre-compile entry (e.g. CompilePromptCard), leave the DailyPageEntryCard post-compile branch in place and move the pre-compile entry to a sticky footer. Do NOT keep the card as a dual-state widget — parent conditionals stay cleaner.
- Mount sticky chrome (compile footer, etc.) as a sibling of the ScrollView inside the outer VStack, ABOVE the InputBarView, so the keyboard and input bar layout are untouched.
- Build command: `xcodebuild -scheme DayPage -destination 'platform=iOS Simulator,arch=arm64,id=3A661BD8-5F57-42B2-B4BA-DBB2FAB4FAD0' build`
- SourceKit diagnostics show macOS errors for AVAudioSession — these are iOS-only APIs and build fine in the iOS target
- SourceKit also misreports "Cannot find X in scope" for cross-module symbols — build succeeds regardless
- VoiceService is @MainActor singleton; reuse VoiceService.transcribe(audioURL:) for the new transcribe-on-swipe-left path (US-008)
- DSColor / DSFonts / DSRadius / DSSpacing tokens used for all visuals
- VaultInitializer.vaultURL is the entry point for vault file paths
- To add new Swift files: add PBXBuildFile + PBXFileReference + group child + Sources entry in project.pbxproj (use sequential IDs)
- DayPageLogger uses nonisolated static helpers called from a DispatchQueue to avoid @MainActor actor isolation issues
- DashScope OpenAI-compatible endpoint is `https://dashscope.aliyuncs.com/compatible-mode/v1` (NOT `coding.dashscope.aliyuncs.com/v1` — that returns 404). The `Secrets.dashScopeBaseURL` env override still wins; only the in-code default changed.
- Use `#if DEBUG print(...)` (not DayPageLogger) for noisy network diagnostics — keeps release logs clean and avoids @MainActor isolation hops.
- For archive-style detail screens that read vault files: load asynchronously inside a `.task` closure into an explicit `LoadState` enum, not inside computed properties. Computed-property IO runs on every re-render and hides failure paths.
- Validate user-supplied date strings with BOTH a strict `^\d{4}-\d{2}-\d{2}$` regex AND a non-lenient DateFormatter round-trip — the regex alone lets `2020-02-30` through.
- When a file-backed view (RawMemoView, DailyPageView) is opened, double-check `FileManager.default.fileExists(atPath:)` inside its own load path even if the parent already checked. Log `url.path` + `errno` on any miss — both are necessary to diagnose sandbox / migration issues.
- When adding a "test this endpoint" Settings action, ALWAYS resolve baseURL/model the same way the production service does (`Secrets.*.isEmpty ? defaultLiteral : Secrets.*`). If you hardcode a URL in Settings, you will silently drift from CompilationService and the test can green while real compile 404s.
- NSURLError codes (timeout=-1001, notConnected=-1009, cannotFindHost=-1003, cannotConnect=-1004) are the right bucket for network-layer hints; HTTP status codes only show up when the server actually responded.
- "Single source of truth for in-flight UI state": when collapsing redundant indicators, also delete the timed `Task` chains that only existed to refresh the now-removed banner. They become dead code (`kind == .progress` guards never fire), and leaving them in the file plants traps for future readers. Either fully delete or pipe their signal into the surviving indicator's view-model field.
- BannerCenter is the right channel for **terminal** outcomes (success / offline / error) but a poor fit for in-flight progress — it competes with timeline cards. Reserve it for "the operation is done, here is the verdict" messages.
- For off-main vault scans, put the real work in a `fileprivate enum`'s `static func` at file scope and call it from inside `Task.detached { ... }.value`. Instance methods fail to compile (`self` capture) and `Self.foo` also fails in detached contexts — file-scope statics are the only clean escape hatch.
- Calendar cell "has data" state should be derived from a pre-scanned `Set<String>` of date basenames (cheap `.contains`), NOT from per-day `DayStats` aggregations (which read file *contents*). Split the two axes: "is anything here?" (pre-scan) vs. "how much activity?" (heatmap aggregation).
- When a detail view already has an explicit `LoadState` handling `.empty`/`.error`/etc, parent cell/list taps should push unconditionally and delete any pre-tap `.alert` guards. Alerts that were protecting a crash-prone detail view become dead UX once the detail view is hardened.
- Feature-flag toggles whose PRD copy is phrased against the legacy path (e.g. '使用旧版输入框') should keep the storage key positive (`useInputBarV2 = true` by default) and invert only inside the Settings Toggle's Binding (`get: { !key }, set: { key = !$0 }`). Avoids 'off means on' confusion later.
- For local popover/menu tap-outside dismissal bounded to a parent view (InputBar, card), a `Color.black.opacity(0.001).contentShape(Rectangle()).onTapGesture { close() }` layer gated by the open-state Bool is enough — no need for a full-screen overlay or DragGesture. Put it in `.background` so it sits behind the popover but in front of the parent view's main content.
- `.photosPicker(isPresented:selection:matching:photoLibrary:)` is the iOS-16 "imperative" API — trigger the picker from any callback closure without having to embed a `PhotosPicker` label view. Pair with `onChange(of: photosPickerItem)` to forward the selection.
- For press-to-talk gestures, use `DragGesture(minimumDistance: 0)` + `@GestureState` for auto-reset, BUT capture the latest translation into a separate `@State CGSize` inside `.onChanged` — some iOS versions reset the gesture's translation before `.onEnded` fires, so reading from `value.translation` in `.onEnded` can give `.zero`. Fall back to the last captured translation when `value.translation == .zero`.
- Fire first-edge haptics (press-down) inside `.onChanged`'s first invocation by gating on `currentPhase == .idle` then transitioning to `.recording`. `.onChanged` has no "did begin" hook — the first call is the begin.
- Attach press-to-talk gestures via `.highPriorityGesture(...)` (not `.gesture` / `.simultaneousGesture`) so an outer ScrollView's DragGesture doesn't steal the touch. `.simultaneousGesture` would let BOTH fire, causing the ScrollView to scroll while the user is holding the button.
- When two gesture thresholds are crossed simultaneously (e.g. diagonal up-left), pick the "safer" outcome as the tiebreaker. For press-to-talk: cancel wins over transcribe when `|dy| >= |dx|` — users expect "discard" on ambiguous gestures, not "send the transcript".
- For a floating in-input-bar overlay that tracks live audio, mount it as the FIRST child of the InputBar's VStack, gated by gesture-phase state. This keeps the overlay above the text field but anchored to the input bar so `ignoresSafeArea` and keyboard rules don't need any changes.
- The transcribe-only path must `try? FileManager.default.removeItem(at: result.fileURL)` + `voiceService.reset()` after reading `result.transcript` — otherwise the audio file lingers in `vault/raw/assets/` and the Whisper queue might retry it later. Only the send path should keep the audio file.
- To unit-test a SwiftUI view's load state machine, extract the pure resolver as a `static func` on the view that takes all dependencies (vaultURL, FileManager) as parameters and returns the state. The `.task { await load() }` hook then just calls the resolver inside `Task.detached` and copies the result into `@State`. You get unit-testable state without any SwiftUI / rendering harness.
- Add a test injection point on singletons by adding an internal `static var testOverrideURL: URL?` that `vaultURL` checks first. `@testable import DayPage` lets tests set/tearDown the override. Don't gate it on `#if DEBUG` — `@testable` already implies a Debug build, and conditional compilation makes the setter invisible from release tests if you ever add them.
- XCTest `testCompiledState_whenBothRawAndDailyExist`-style names read like specs. Pattern: `test<Outcome>_when<Condition>`. Makes `xcodebuild test` output self-documenting.
- Manually adding an iOS unit-test target to a hand-maintained `project.pbxproj`: you need PBXBuildFile + PBXFileReference (source + xctest product in Products group), a new PBXGroup as a child of the project mainGroup (NOT inside the app group — same level as `DayPage`), PBXNativeTarget with productType `com.apple.product-type.bundle.unit-test`, PBXSourcesBuildPhase + PBXFrameworksBuildPhase (empty `files = ()` is fine; XCTest gets linked via test-host), PBXTargetDependency + PBXContainerItemProxy (proxyType=1 on the app target), XCBuildConfigurations with `TEST_HOST = $(BUILT_PRODUCTS_DIR)/DayPage.app/$(BUNDLE_EXECUTABLE_FOLDER_PATH)/DayPage` + `BUNDLE_LOADER = $(TEST_HOST)`, XCConfigurationList, added to `targets = ( ... )` in PBXProject and to `TargetAttributes` with `TestTargetID = <app-target-id>`. Finally, update the scheme's BuildAction (add the xctest buildForTesting="YES") and TestAction (add a `<Testables><TestableReference><BuildableReference .../>` block).
- Use ID prefix `T` for test-target-specific pbxproj objects (T1*, T2*, T3*, T4*, T5*, T7*, T8*, T9*, TC*) to keep them visually grouped from the app's `A*` IDs. Makes the pbxproj diff-friendly in future.

- **Learnings for future iterations:**
  - Want off-main work that doesn't capture `self`? Move the real work into a `fileprivate enum`'s `static func`. Instance methods, `Self.foo`, and even `self.foo` all fail inside `Task.detached`'s non-isolated closure; file-scope statics are the only clean escape hatch.
  - When pre-scanning a vault directory, list the contents (`contentsOfDirectory(atPath:)`) and parse basenames — do NOT read file contents. One directory listing is O(1) IOs; reading 365 daily files on appear would stall the tab-switch animation.

  - Two `@State Set<String>`s beats a single `[String: CellDataState]` dictionary — Sets are cheaper to populate off-main, and `.contains(...)` is the only query the view makes.
  - The pre-scan scans the entire vault (not just the current month). Month navigation doesn't re-scan — cells for past/future months read from the same sets. This is why it's important to re-trigger `preScanVault()` on every `onAppear`, not just the first: a compile on Today tab adds a new `vault/wiki/daily/YYYY-MM-DD.md`, and coming back to Archive should surface it without restart.

- **Learnings for future iterations:**
  - `DragGesture(minimumDistance: 0)` is the only clean way to get both press-down and release in one gesture recognizer. Tap + LongPress + DragGesture combined via `.simultaneously(with:)` creates race conditions — the minimumDistance-0 drag handles everything.
  - `@GestureState` resets automatically when the gesture ends, but reading `value.translation` in `.onEnded` can return `.zero` on some iOS builds. Always keep a separate `@State lastTranslation: CGSize` that you update in every `.onChanged` and fall back to it on release.
  - `.highPriorityGesture` is the correct attachment for press-to-talk inside a scrollable parent. `.gesture` and `.simultaneousGesture` let the outer ScrollView's DragGesture fire too, causing scroll-during-hold bugs.
  - Haptic feedback is best done with `UIImpactFeedbackGenerator(style:)` created per-edge with `.prepare()` called just before `.impactOccurred()`. Don't try to reuse a single generator across phases — the warm-up window is part of the API contract, and reusing can drop haptics on fast transitions.
  - When a gesture has multiple armed states (cancel / transcribe), pick the "safer" outcome as the tiebreaker for ambiguous diagonals. Here: cancel wins when `|dy| >= |dx|`. Users get upset when a slightly-up-and-left swipe sends a transcript instead of cancelling.
  - For an overlay that needs live data from a `@StateObject` service, prefer putting the `@StateObject` on the view that mounts the overlay (InputBarV2) rather than passing the service down through the callback shape. The parent-view callbacks stay pure (result in / transcript in) and the overlay reads directly from the service via SwiftUI's binding.
  - The transcribe-only path must explicitly delete the audio file (`FileManager.default.removeItem(at:)`) because `VoiceService.stopAndTranscribe()` writes it to `vault/raw/assets/` for the send path to consume. If you skip the delete, the next `VoiceAttachmentQueue` pass may retry the orphaned audio file.

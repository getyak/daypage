## Codebase Patterns (carried over from Wave 2)
- isAttachmentDownloaded: read .ubiquitousItemDownloadingStatusKey via URL.resourceValues; non-ubiquitous files return true (no iCloud key present)
- pollDownloadStatus: use Task { for _ in 0..<30 { await Task.sleep(2s); if downloaded { insert to Set; return } } } pattern for polling without timers in SwiftUI views
- AppSettings nonisolated accessors (e.g., currentTimeZone, currentAttachmentPolicy, currentVaultLocation): pattern for reading UserDefaults from non-MainActor contexts like VaultInitializer static methods
- VaultMigrationService.verifyMigration is internal (not private) to enable direct unit testing from DayPageTests
- @MainActor class methods called from XCTest async tests: wrap with await MainActor.run { ... } — XCTAssertThrowsError doesn't work for @MainActor methods, use do/catch inside MainActor.run instead
- PhotoDownloadPlaceholder / AudioDownloadPlaceholder: standalone struct views for iCloud eviction states; keep outside MemoCardView to allow reuse
- ConflictMerger uses NSFileVersion.unresolvedConflictVersionsOfItem (not NSMetadataQuery) for conflict detection — simpler and works without a running query
- NSFileVersion conflict resolution: set version.isResolved = true, then call NSFileVersion.removeOtherVersionsOfItem — both steps required
- ConflictMerger.resolveConflictsIfNeeded(in:) can be called from VaultInitializer or app foreground to sweep for any unresolved conflicts
- Test IDs in pbxproj: T20000002 (file ref), T10000002 (build file) — next available: T20000003 / T10000003
- VaultInitializer.vaultURL is the single entry point for all 37 vault file path accesses
- RawStorage.atomicWrite uses temp-file + FileManager.replaceItemAt pattern for atomic writes
- Services are @MainActor final class singletons; view models are @MainActor final class: ObservableObject
- DSColor / DSFonts / DSRadius / DSSpacing tokens used for all visuals
- To add new Swift files: add PBXBuildFile + PBXFileReference + group child + Sources entry in project.pbxproj (use sequential IDs A1xxxxxxx / A2xxxxxxx, last used A20000056)
- Build command: xcodebuild -scheme DayPage -destination 'platform=iOS Simulator,arch=arm64,id=3A661BD8-5F57-42B2-B4BA-DBB2FAB4FAD0' build
- Test command: xcodebuild -scheme DayPage -destination 'platform=iOS Simulator,arch=arm64,id=3A661BD8-5F57-42B2-B4BA-DBB2FAB4FAD0' test
- SourceKit diagnostics show macOS/cross-module false positives — trust xcodebuild, not SourceKit
- DayPageTests target uses T* pbxproj ID prefix; app uses A* prefix
- VaultInitializer.testOverrideURL is the injection point for unit tests
- RawStorage.atomicWrite is now wrapped in NSFileCoordinator; temp-file + replaceItemAt logic runs inside the coordinator block
- iCloudVaultLocator.vaultURL falls back to LocalVaultLocator().vaultURL when iCloud unavailable (nil ubiquity container)
- VaultInitializer.shared is a lazy static var (closure-evaluated once at first access); safe to reassign later for rollback

## Codebase Patterns
- Build command: `xcodebuild -scheme DayPage -destination 'platform=iOS Simulator,arch=arm64,id=3A661BD8-5F57-42B2-B4BA-DBB2FAB4FAD0' build`
- SourceKit diagnostics show macOS errors for AVAudioSession — these are iOS-only APIs and build fine in the iOS target
- SourceKit also misreports "Cannot find X in scope" for cross-module symbols (RawStorage, VaultInitializer, etc.) — build succeeds regardless
- VoiceService is @MainActor singleton; VoiceRecordingView uses @StateObject private var voiceService = VoiceService.shared
- DSColor tokens used for all colors in existing views
- VaultInitializer.vaultURL is the entry point for vault file paths
- To add new Swift files: add PBXBuildFile + PBXFileReference + group child + Sources entry in project.pbxproj (use sequential IDs like A10000036/A20000036)
- DayPageLogger uses nonisolated static helpers called from a DispatchQueue to avoid @MainActor actor isolation issues

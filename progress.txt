## Codebase Patterns (carried over from Wave 1, append new findings here)
- Track ScrollView scroll position with a named `.coordinateSpace(name:)` on the ScrollView + a 1pt `GeometryReader`-backed anchor view at the end of its content that emits its `.frame(in: .named(...)).minY` via a `PreferenceKey`. Parent reads both the anchor minY and the ScrollView's visible height (from the outer GeometryReader's `geo.size.height`) to derive "near bottom" — no ScrollViewReader needed for that. Works on iOS 16.
- When removing a card from the timeline that was used as the pre-compile entry (e.g. CompilePromptCard), leave the DailyPageEntryCard post-compile branch in place and move the pre-compile entry to a sticky footer. Do NOT keep the card as a dual-state widget — parent conditionals stay cleaner.
- Mount sticky chrome (compile footer, etc.) as a sibling of the ScrollView inside the outer VStack, ABOVE the InputBarView, so the keyboard and input bar layout are untouched.
- Build command: `xcodebuild -scheme DayPage -destination 'platform=iOS Simulator,arch=arm64,id=3A661BD8-5F57-42B2-B4BA-DBB2FAB4FAD0' build`
- SourceKit diagnostics show macOS errors for AVAudioSession — these are iOS-only APIs and build fine in the iOS target
- SourceKit also misreports "Cannot find X in scope" for cross-module symbols — build succeeds regardless
- VoiceService is @MainActor singleton; reuse VoiceService.transcribe(audioURL:) for the new transcribe-on-swipe-left path (US-008)
- DSColor / DSFonts / DSRadius / DSSpacing tokens used for all visuals
- VaultInitializer.vaultURL is the entry point for vault file paths
- To add new Swift files: add PBXBuildFile + PBXFileReference + group child + Sources entry in project.pbxproj (use sequential IDs)
- DayPageLogger uses nonisolated static helpers called from a DispatchQueue to avoid @MainActor actor isolation issues
- DashScope OpenAI-compatible endpoint is `https://dashscope.aliyuncs.com/compatible-mode/v1` (NOT `coding.dashscope.aliyuncs.com/v1` — that returns 404). The `Secrets.dashScopeBaseURL` env override still wins; only the in-code default changed.
- Use `#if DEBUG print(...)` (not DayPageLogger) for noisy network diagnostics — keeps release logs clean and avoids @MainActor isolation hops.
- For archive-style detail screens that read vault files: load asynchronously inside a `.task` closure into an explicit `LoadState` enum, not inside computed properties. Computed-property IO runs on every re-render and hides failure paths.
- Validate user-supplied date strings with BOTH a strict `^\d{4}-\d{2}-\d{2}$` regex AND a non-lenient DateFormatter round-trip — the regex alone lets `2020-02-30` through.
- When a file-backed view (RawMemoView, DailyPageView) is opened, double-check `FileManager.default.fileExists(atPath:)` inside its own load path even if the parent already checked. Log `url.path` + `errno` on any miss — both are necessary to diagnose sandbox / migration issues.
- When adding a "test this endpoint" Settings action, ALWAYS resolve baseURL/model the same way the production service does (`Secrets.*.isEmpty ? defaultLiteral : Secrets.*`). If you hardcode a URL in Settings, you will silently drift from CompilationService and the test can green while real compile 404s.
- NSURLError codes (timeout=-1001, notConnected=-1009, cannotFindHost=-1003, cannotConnect=-1004) are the right bucket for network-layer hints; HTTP status codes only show up when the server actually responded.
- "Single source of truth for in-flight UI state": when collapsing redundant indicators, also delete the timed `Task` chains that only existed to refresh the now-removed banner. They become dead code (`kind == .progress` guards never fire), and leaving them in the file plants traps for future readers. Either fully delete or pipe their signal into the surviving indicator's view-model field.
- BannerCenter is the right channel for **terminal** outcomes (success / offline / error) but a poor fit for in-flight progress — it competes with timeline cards. Reserve it for "the operation is done, here is the verdict" messages.
- For off-main vault scans, put the real work in a `fileprivate enum`'s `static func` at file scope and call it from inside `Task.detached { ... }.value`. Instance methods fail to compile (`self` capture) and `Self.foo` also fails in detached contexts — file-scope statics are the only clean escape hatch.
- Calendar cell "has data" state should be derived from a pre-scanned `Set<String>` of date basenames (cheap `.contains`), NOT from per-day `DayStats` aggregations (which read file *contents*). Split the two axes: "is anything here?" (pre-scan) vs. "how much activity?" (heatmap aggregation).
- When a detail view already has an explicit `LoadState` handling `.empty`/`.error`/etc, parent cell/list taps should push unconditionally and delete any pre-tap `.alert` guards. Alerts that were protecting a crash-prone detail view become dead UX once the detail view is hardened.

## 2026-04-17 - US-006 Archive 日历区分有数据 / 无数据日期
- `DayPage/Features/Archive/ArchiveView.swift`: added two `@State Set<String>` — `rawDates`, `dailyDates`. On `.onAppear`, `viewModel.loadMonth()` still runs (keeps monthly summary heatmap intact) and a sibling `Task { await preScanVault() }` kicks off an off-main scan of `vault/raw/*.md` and `vault/wiki/daily/*.md` via `Task.detached(priority: .utility)`. Results published back on MainActor into the two sets.
- Pre-scan parsing lives in a new `fileprivate enum ArchiveVaultScan { static func listDateFilenames(in:fileManager:) -> Set<String> }` at file scope. Static helper + file-scope enum means the `Task.detached` closure can call it without capturing `self` or tripping `actor-isolated` errors — calling `Self.foo` or `someInstanceMethod` in a detached Task both fail. Parsing filters to `*.md` then validates basename length == 10 + dashes at offsets 4/7 + 8 digits, dropping any stray files (assets/, attachments, macOS .DS_Store, etc.).
- New private `CellDataState` enum (`.compiled` / `.rawOnly` / `.none`) derived from the sets in `cellState(for:)`. Calendar cell rewritten: `.compiled` → `DSColor.primary` fill + `DSColor.onPrimary` text; `.rawOnly` → `DSColor.surfaceContainerLow` fill + `DSColor.onSurface` text + 4pt bottom-trailing dot; `.none` → 50% translucent `surfaceContainerLow` fill + 50% translucent `onSurface` text. Today retains the 2pt primary stroke across all three states. Added accessibility labels for each state so VoiceOver announces "date, 已编译" / "有原始记录" / "无记录".
- `handleDateTap(dateStr:stats:)` collapsed to `handleDateTap(dateStr:)` — every cell now unconditionally `showDayDetail = true`. The old `showNoRecordAlert` and `.alert(...)` modifier are gone entirely; DayDetailView (US-002) already handles the `.empty` / `.error` paths for invalid or missing-data dates. Also updated the two other call sites (filteredDays list row inside monthlySummary and archiveListRow in list mode) to the single-argument signature.
- Files changed: `DayPage/Features/Archive/ArchiveView.swift`, `prd.json`, `progress.txt`.
- Build: `xcodebuild -scheme DayPage ... build` → **BUILD SUCCEEDED**.
- Manual Simulator verification of three-state visual + tap-through on no-data cells deferred (`verify-daypage` MCP skill not wired). The state machine is pure data-flow: sets populate → cellState derives → three branches in ViewBuilder. No timing, no animation races.
- **Learnings for future iterations:**
  - Want off-main work that doesn't capture `self`? Move the real work into a `fileprivate enum`'s `static func`. Instance methods, `Self.foo`, and even `self.foo` all fail inside `Task.detached`'s non-isolated closure; file-scope statics are the only clean escape hatch.
  - When pre-scanning a vault directory, list the contents (`contentsOfDirectory(atPath:)`) and parse basenames — do NOT read file contents. One directory listing is O(1) IOs; reading 365 daily files on appear would stall the tab-switch animation.
  - `CellDataState` from pre-scanned sets is semantically different from `DayStats.DensityLevel` (which counts memos for the heatmap). Keep them separate: the calendar cell cares about "is there anything here?" while the monthly summary heatmap cares about "how much activity." Forcing them into one enum bloats both.
  - When a detail view (DayDetailView) already handles all failure modes, the parent should just push unconditionally. The old "guard the tap with an alert" pattern predates US-002's state machine and became dead UX — one of those cases where removing code is the feature.
  - Two `@State Set<String>`s beats a single `[String: CellDataState]` dictionary — Sets are cheaper to populate off-main, and `.contains(...)` is the only query the view makes.
  - The pre-scan scans the entire vault (not just the current month). Month navigation doesn't re-scan — cells for past/future months read from the same sets. This is why it's important to re-trigger `preScanVault()` on every `onAppear`, not just the first: a compile on Today tab adds a new `vault/wiki/daily/YYYY-MM-DD.md`, and coming back to Archive should surface it without restart.
